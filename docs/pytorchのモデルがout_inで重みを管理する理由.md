はい、ここは最初かなり引っかかるポイントです。
でも理由はそんなに難しくなくて、**「出力ニューロンごとに1行で重みを持つと都合がいいから」**です 😊
---

# 結論

PyTorch が重みを

```text
(out_features, in_features)
```

の形で持つのは、**1個の出力を作るための重みを1行にまとめたいから**です。

なので内部的には

```text
y = x @ W.T + b
```

の形で計算されます。

---

# まず、なぜそうなるのか

たとえば全結合層を考えます。

* 入力次元: 3
* 出力次元: 2

だとします。

このとき、出力は2個あります。

```text
y1 = w11*x1 + w12*x2 + w13*x3 + b1
y2 = w21*x1 + w22*x2 + w23*x3 + b2
```

ここで大事なのは、

* `y1` を作る重みは1セット
* `y2` を作る重みは1セット

ということです。

つまり、**出力1個につき重みベクトル1本** 持っています。

だから行列で並べると、

```text
W =
[
  [y1用の重み]
  [y2用の重み]
]
```

となるのが自然です。

つまり shape は

```text
(出力数, 入力数) = (out_features, in_features)
```

です。

---

# たとえ話

これは「テストの採点表」みたいなものです ✍️

* 列: 入力項目
* 行: 出力ニューロン

各行には
**その出力ニューロンが、各入力をどれだけ重視するか**
が並んでいます。

だから

* 1行目 = 出力1番のルール
* 2行目 = 出力2番のルール

という見方になります。

この形だと、「この出力はどの重みで作られるのか」がすごく見やすいです。

---

# 数学の式ではなぜ `W x` と書けるの？

数学では入力ベクトルを **縦ベクトル** で書くことが多いです。

```text
x: (in_features, 1)
W: (out_features, in_features)
```

なので

```text
W x
```

でそのまま計算できます。

結果は

```text
(out_features, 1)
```

になります。

---

# PyTorchではなぜ `x @ W.T` になるの？

PyTorch では入力 `x` をたいてい **横向きに並べたバッチ** として持ちます。

```text
x.shape = (batch_size, in_features)
```

たとえば

```text
x.shape = (32, 10)
W.shape = (5, 10)
```

のとき、そのまま

```text
x @ W
```

はできません。

なぜなら

```text
(32, 10) @ (5, 10)
```

で、内側の `10` と `5` が合わないからです。

そこで転置して

```text
W.T.shape = (10, 5)
```

にすると

```text
(32, 10) @ (10, 5) = (32, 5)
```

となって、バッチごとの出力が一気に計算できます。

だから PyTorch では

```text
x @ W.T + b
```

が自然になります。

---

# じゃあ最初から `(in, out)` で持てばいいのでは？

これも自然な疑問です。
でも `(out, in)` にしておくと、**各出力ユニットの重みを1行で管理できる**ので扱いやすいです。

たとえば出力3番の重みだけ見たいなら、

```python
weight[2]
```

でそのまま取れます。

もし `(in, out)` で持つと、出力ごとの重みは列に散らばるので、少し見づらいです。

---

# LSTMでも同じ考え方

LSTM の `weight_ih_l0` も同じです。

```text
weight_ih_l0.shape = (4 * hidden_size, input_size)
```

これは

* 入力ゲート用
* 忘却ゲート用
* 候補メモリ用
* 出力ゲート用

の **4種類の出力をまとめて作る** 重みです。

つまりここでも、
**「出力側ごとに行を持つ」**
という形になっています。

だから内部計算はイメージとして

```text
x_t @ weight_ih_l0.T
```

になります。

---

# すごくざっくりまとめると

PyTorch が `(out, in)` にしているのは、

## 1.

**1つの出力に対応する重みを1行にまとめられてわかりやすいから**

## 2.

**数学の式 `W x` と相性がいいから**

## 3.

**バッチ入力では `x @ W.T` にすれば一気に計算しやすいから**

です。

---

**「出力を作るためのルールを、出力ごとに1行ずつ並べて保存している」**
と思うとかなりわかりやすいです。
