`nn.CrossEntropyLoss()` は、**分類問題でよく使う損失関数**です。
ざっくりいうと、

> **モデルの予想と正解ラベルがどれくらいズレているか**
> を数字で表すものです 📉

---

## まず結論：target に入る値

`target` には **正解クラスの番号** を入れます。
つまり、**one-hotベクトルではなく整数ラベル**です。

たとえば3クラス分類なら、

* クラス0
* クラス1
* クラス2

のように番号をつけて、`target` にはその番号を入れます。

例：

```python
target = torch.tensor([2, 0, 1])
```

これは

* 1個目のデータの正解はクラス2
* 2個目のデータの正解はクラス0
* 3個目のデータの正解はクラス1

という意味です。

---

## イメージで理解する

3択クイズを想像してください 🎯

モデルが

* Aが正解っぽい
* Bが正解っぽい
* Cが正解っぽい

と点数を出します。

でも本当の正解は1つだけです。
その**正解の番号**を `target` に入れる感じです。

なので `target` は、

```python
[正解A, 正解B, 正解C]
```

ではなく、

```python
[0, 1, 2]
```

のような**クラス番号**になります。

---

## 基本的な使い方

```python
import torch
import torch.nn as nn

loss_fn = nn.CrossEntropyLoss()

output = torch.tensor([
    [2.0, 0.5, 0.3],
    [0.1, 1.2, 2.1]
])  # モデルの出力

target = torch.tensor([0, 2])  # 正解ラベル

loss = loss_fn(output, target)
print(loss)
```

---

## output には何を入れる？

`output` には **softmaxをかける前の生の値** を入れます。
これを **logits** と呼びます。

ここがかなり大事です。

### 正しい例

```python
output = model(x)   # softmaxしない
loss = loss_fn(output, target)
```

### よくある間違い

```python
output = torch.softmax(model(x), dim=1)  # ← 先にsoftmaxしてしまう
loss = loss_fn(output, target)
```

`CrossEntropyLoss` の中で、実質的に **Softmax + log + 誤差計算** をまとめてやってくれるので、
**自分でsoftmaxしない**のが基本です。

---

## shape はどうなる？

### 2次元の分類なら

`output` の形：

```python
(batch_size, クラス数)
```

`target` の形：

```python
(batch_size,)
```

例：

```python
output.shape = (4, 3)
target.shape = (4,)
```

これは

* 4個のデータがあって
* 各データは3クラス分類

という意味です。

---

## target に one-hot を入れてはいけないの？

基本形では **ダメ** です。

たとえば3クラス分類で、正解がクラス1なら

### one-hot

```python
[0, 1, 0]
```

ではなく、

### CrossEntropyLossで使う target

```python
1
```

を入れます。

---

## 具体例

### 3クラス分類

```python
import torch
import torch.nn as nn

loss_fn = nn.CrossEntropyLoss()

output = torch.tensor([
    [1.2, 0.3, -0.5],
    [0.1, 2.0, 0.4],
    [0.2, 0.1, 3.1]
])

target = torch.tensor([0, 1, 2])

loss = loss_fn(output, target)
print(loss)
```

この場合、

* 1行目の正解はクラス0
* 2行目の正解はクラス1
* 3行目の正解はクラス2

です。

---

## target の dtype にも注意

`target` は通常 **整数型 `torch.long`** です。

```python
target = torch.tensor([0, 1, 2], dtype=torch.long)
```

`float` にするとエラーになることがあります。

---

## 画像分類以外でも使える？

使えます。
**「候補の中からどれが正解か選ぶ問題」** ならかなり広く使われます。

たとえば

* 画像分類
* テキスト分類
* 単語予測
* 時系列のクラス分類

などです。

---

## 系列データではどうなる？

たとえば文章生成では、各時刻ごとに「次の単語は何か」を当てます。

このときも考え方は同じで、
`target` には **正解単語のID** を入れます。

例：

```python
target = torch.tensor([5, 10, 2, 8])
```

これは単語番号です。

つまり、画像分類での「猫=0, 犬=1」と同じで、
NLPでは「hello=5, world=10」のような**語彙ID**が入ります。

---

## まとめ

### `nn.CrossEntropyLoss()` の役割

* 分類問題の誤差を計算する

### `output` に入れるもの

* **softmax前の生の出力(logits)**

### `target` に入れるもの

* **正解クラス番号の整数**
* one-hotではない

### 例

```python
output = torch.tensor([[2.1, 0.3, 1.0],
                       [0.2, 1.5, 0.1]])
target = torch.tensor([0, 1])
```

---

## 覚え方のコツ

`CrossEntropyLoss` は
**「答えの番号を渡す損失関数」**
と覚えるとかなりわかりやすいです 👍

* モデル出力 → 各クラスの点数
* target → 正解の番号

必要なら次に
**「なぜ softmax しなくてよいのか」**
を図っぽいイメージでわかりやすく説明できます。
